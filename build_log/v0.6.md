📜 Build Log v0.6 – 
    File & Folder Operations Update

🚀 Major Feature Expansion
    This update significantly expands the File Manager from a simple file browser into a complete terminal-based    
    file management application by introducing full file and folder manipulation capabilities.

📁 Folder Operations
    Added Create Folder functionality.
    Added Rename Folder support.
    Added Copy Folder using recursive directory copying.
    Added Move Folder functionality.
    Added Delete Folder functionality.
    Added automatic duplicate folder naming (Folder(1), Folder(2), etc.) when copying to the same destination.

📄 File Operations
    Added Create File functionality with support for:
    Text Documents (.txt)
    PDF Documents (.pdf)
    Microsoft Word Documents (.docx)
    Microsoft Excel Worksheets (.xlsx)
    Microsoft PowerPoint Presentations (.pptx)
    Added Rename File support.
    Added Copy File functionality.
    Added Move File functionality.
    Added Delete File functionality.
    Added automatic duplicate filename generation to prevent overwriting existing files.

📂 Destination Selection
    Implemented a dedicated destination selection interface for copy and move operations.
    Destination browser displays folders only, preventing accidental file selection.
    The destination selector now opens directly in the source file's parent directory instead of always starting from the drive root.
    Added navigation through folders, parent directories, and drive switching.

🛡 Validation & Error Handling
    Prevented creation of files and folders with invalid names.
    Blocked Windows-invalid filename characters.
    Prevented duplicate file/folder creation.
    Added handling for:
    Permission errors
    Invalid paths
    Existing files/folders
    General filesystem errors

🎨 User Experience Improvements
    Added clear success and error messages for every operation.
    Improved menu layouts for file and folder operations.
    Added confirmation prompts where appropriate.
    Enhanced the terminal interface with colors and emojis for better readability.

🔧 Internal Improvements
    Refactored the project by separating the navigation logic into a dedicated nav.py module while keeping filesystem operations inside utils.py, resulting in a cleaner and more maintainable codebase.
