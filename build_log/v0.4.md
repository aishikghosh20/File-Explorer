Build Log — v0.4.0

Release: File Opening Support

Overview

This release transforms the project from a navigation-focused terminal explorer into an interactive file explorer capable of launching supported files using their default Windows applications.

---

✨ New Features

File Opening

- Added support for opening compatible files directly from the explorer.
- Files can now be selected directly from the directory listing.
- Integrated with the Windows default application handler.

Supported File Types

- Text files (".txt")
- PDF documents (".pdf")
- Microsoft Office documents (".doc", ".docx", ".xls", ".xlsx", ".ppt", ".pptx")
- Images (".jpg", ".jpeg", ".png", ".gif", ".bmp", ".webp")
- Audio (".mp3", ".wav", ".flac", ".aac")
- Videos (".mp4", ".mkv", ".avi", ".mov")
- Archives (".zip", ".rar", ".7z")
- Python files (".py")
- Executables (".exe", ".msi")

---

🔒 Safety Improvements

Executable Confirmation

Before launching an executable, the explorer now displays a confirmation prompt to reduce accidental execution.

- Run executable
- Cancel operation

---

⚙️ Internal Improvements

- Created a dedicated "open_file()" helper function.
- Added a dedicated "open_exe()" helper using "subprocess.Popen()".
- Explorer now waits for launched executables to terminate before returning to the navigation menu.
- Improved exception handling for file launching operations.
- Unsupported file types are detected before attempting to open them.

---

🐞 Fixes

- Prevented crashes when opening unsupported files.
- Improved handling of file launch failures.
- Preserved the current directory after opening or cancelling a file operation.
- Improved navigation flow after file execution.
