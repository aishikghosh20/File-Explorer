Build Log — v0.4 "File Management"

Release Date: 29 June 2026

✨ New Features
    📝 File Rename
        Added dedicated Rename File option.
        Users can select a file from the current directory.
        Added confirmation menu before renaming.
        Supports reselecting another file before confirming.
        Successfully renames while preserving the original extension.
        Displays both old and new filenames after a successful rename.

    🗑️ File Delete
        Added dedicated Delete File option.
        Users can select a file from the current directory.
        Added confirmation prompt before permanent deletion.
        Supports selecting another file before deleting.
        Permanently deletes files from disk.
        Returns safely to the directory view after deletion.

🛡️ Validation Improvements
    Rename Validation
    Prevents empty filenames.
    Prevents invalid Windows filename characters:
    < > : " / \ | ? *
    Detects duplicate filenames.
    Handles permission errors gracefully.
    Displays meaningful error messages.
    Delete Validation
    Confirmation required before deletion.
    Handles permission errors.
    Handles missing files.
    Handles unexpected OS errors safely.

🎨 User Experience
    Improved confirmation dialogs.
    Cleaner navigation flow.
    Better success and error messages.
    Cancel option added to file selection.
    Ability to choose another file before confirming actions.
    Automatic return to the directory menu after operations.

🔧 Internal Improvements
    Reused helper functions for cleaner code.
    Reduced duplicate logic.
    Improved control flow.
    Better exception handling.
    More consistent menu behavior.
